Pierre Béhel @ 2014-05-19 22:05:00

Une adresse reste indispensable pour retrouver un élément. Pour retrouver un livre, vous avez ainsi un annuaire des livres, le FEL (Fichier Exhaustif des Livres). Le problème n’est pas l’adresse mais qui maîtrise ce que l’on peut retrouver, autrement dit qui possède le système d’adressage, qui le contrôle. Déjà le système DNS (sous contrôle américain) est un problème de cette nature, bien avant Google ou Facebook.

Je vous suggère de vous pencher sur Emenu, un concept d’univers virtuel P2P décentralisé où les registres d’adressage sont également décentralisés http://www.pierrebehel.fr/emenu/

mmai @ 2014-05-23 13:54:00

Une pratique intéressante inspirée du mode de partage décentralisé du code des logiciels libres se développe chez certains blogueurs, il s’agit de placer chaque article sous gestionnaire de version, l’article est alors associé à un identifiant unique (le numéro de commit) et peut être partagé et hébergé n’importe où.

Ainsi l’auteur du blog http://substack.net/ indique le numéro de commit en en-tête de chaque billet, il propose également de cloner la totalité du blog chez soi et de suivre ses mises à jour avec une commande du type "git clone http://substack.net/blog.git".

On retrouve beaucoup de ces blogs sur la plate-forme d’hébergement de code github, par exemple les articles de http://esquisses.clochix.net/ sont également disponibles ici : https://github.com/clochix/esquisses/tree/gh-pages/\_posts

Les connaissances techniques nécessaire à la mise en place de cette architecture la réserve pour le moment à un public restreint, mais je la trouve élégante et prometteuse.
